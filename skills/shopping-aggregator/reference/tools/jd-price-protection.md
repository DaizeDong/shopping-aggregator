# Tool: 京东价保 (JD Price Protection / 价格保护)

- **Domain(s):** jd-pdd
- **Barrier route:** ① 平台官方机制 · **Source tier:** L1（官方退差，自营最稳） · **Ready MCP:** no
- **Cost:** 免费（京东内置服务，无需付费订阅）
- **Repo / Provider:** 京东 help.jd.com 官方价格保护服务 · 帮助页 help.jd.com/user/issue/291-548.html（自营）、291-3785.html（三方）、292-551.html（申请流程）
- **Top pick for its domain:** yes for **"买完之后兜底降价"**（不是比价工具，是下单后的差价退还机制）

## What it does / when to pick it
**不是价格情报源，是京东自家的下单后退差价机制**。买带「价保」标识的商品后（标识形如 `7天价保`/`15天价保`/`30天价保`/`90天价保`/`180天价保`/`365天价保`/`价保618`/`价保双11`，以商品页实际展示为准），在价保周期内同一商品京东价降价的，可申请**退还差价**——不是优惠券、不是历史价曲线，是真金白银退回。

**何时用**：用户已在京东（尤其京东自营）下单、问「买完后降价能不能退差」时，这是首选答案。比价工具（慢慢买/购物党）告诉你「该不该买」；价保告诉你「买完跌价怎么补救」。报告里凡涉及京东订单，应**主动提醒启用价保**——其兜底价值常高于一次性比价。

## Install
无需安装。是京东账号内置功能：京东 App 或网页登录即可用。**大陆京东账号约束**：价保是大陆京东 (jd.com) 的服务，**京东国际 / 跨境页面规则不同**，不要混。

## Auth / keys
无 API key。需**已登录的京东账号** + 该账号名下的有效订单。无开放 API——京东联盟/开放平台**不提供公开价格或价保查询接口**（仅给卖家自家数据），agent 无法编程式批量申请；只能引导用户在 App/网页内操作。

## Usage — call examples
- **用户（网页）**：京东首页右上角【我的京东】→ 左侧【客户服务】→【价格保护】→ 找到订单 → 点【申请】/【申请处理】。
- **用户（App）**：京东 App →【我的】→【客户服务】→【价格保护】→ 选订单申请。
- **agent**：**没有可调用的 API**。agent 的角色是**提醒 + 算差价 + 列资格窗口**，把申请动作交给用户。可帮用户算：`单品价保金额 = 下单时基准价 − 申请价保时基准价 − 已返还金额`，其中 `基准价 = 京东价 − 优惠金额（直降/返现/返京券/满减）`。

## Eligibility window（资格窗口，自营 vs 三方不同）
- **京东自营**（不含厂商配送）：签收前或**签收次日 00:00:00 起** 7/15/30/90/180/365 天内（以价保标识为准）。
- **第三方商家 / 厂商配送**：**下单提交时刻起**算（168/360/720/2160/4320/8760 小时 = 7/15/30/90/180/365 天），不是签收起算。
- **厂商配送商品**：价保有效期自下单之日起算，逐品不同，**以价保申请页实时显示为准**。

## Coverage / 不保的情况（关键——别高估覆盖）
退差方式：审核通过后，货到付款未付按新价付；微信/储蓄卡/云闪付/白条/小金库/信用卡/QQ/数币/ApplePay → 退至钱包余额/小金库；其他 → 退原支付账户。已开增票/已报销抵扣的，差价改走京豆返还。

**不支持价保的常见场景（官方列举）**：超级补贴/月黑风高/超级补贴多人团、直播间专享价、支付优惠（支付券/白条券/白条立减）导致的降价；消费券、国补/政府补贴/焕新补贴、限时补贴（限时立减/折扣/折上折）、惊喜组合优惠、京享礼金、以旧换新、参团、京补合约等活动降价；商品已下架/秒杀中/无货不可预订/抢购/超限购；含赠品订单及赠品本身；无效订单（已取消/已删除）；在线支付未付款。

## General experience & gotchas (踩坑)
- **只算保价期内差价，不给完整历史曲线**——要看上市至今的全程低点，仍需慢慢买。两者互补。
- **「价保标识」才有价保**——没有标识的商品（部分三方）根本不进价保体系；下单前看商品页。
- **大量「补贴类」降价被排除**——百补、国补、限时补贴、直播专享这些恰恰是国内最常见的降价形式，**这些跌价价保不赔**。所以「京东降价了」≠「能价保」，务必对照不保清单。
- **实名认证上限**：同一用户价保累计返还金额有上限，超额需实名认证，未实名无法继续申请。
- **退款联动**：已价保的订单若整单退款，系统先扣回价保金额再退。
- **买贵双倍赔 与 价保 互斥扣减**：同一订单已获其一补偿，申请另一须以「实付 − 已获补偿」为到手价。
- **规则随商品/大促浮动**——「价保618」「价保双11」是临时延长窗口；**一切以下单时商品页 + 申请页实时展示为准**，本 doc 仅为机制说明。

## Failure signals & fallback
申请页显示「该订单不支持价保」「已超出价保期」「商品已下架」、或降价命中不保清单。**Fallback:** 慢慢买（历史价判断是否真低点）、购物党（实时找券，用更优惠券摊低到手价而非退差）、什么值得买（社区蹲更深折扣）。**价保是兜底不是比价**——它替代不了买前的历史价核查。
