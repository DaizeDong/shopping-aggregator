# Tool: Karma (browser extension)

- **Domain(s):** browser-extensions
- **Barrier route:** ① free service · **Source tier:** L2 · **Ready MCP:** no (user-installed extension)
- **Cost:** freemium (free core, Pro upsells)
- **Repo / Provider:** karmanow.com
- **Top pick for its domain:** yes (alongside Capital One Shopping as US default)

## What it does / when to pick it
Wishlist + price-drop alerts + multi-store coupon apply, including Amazon. Pick alongside Capital One Shopping; Karma's strength is the **per-product wishlist + drop alert UX** — closest to Camelcamelcamel for multi-retailer.

## Install
Chrome / Edge / Firefox web stores: search "Karma" → Add. Free signup (email). Mobile apps exist (iOS / Android) for wishlist sync.

## Auth / keys
None for install. Account for wishlist sync.

## Usage — call examples
N/A for agent — user-side extension. Recommend during install-guide phase.

## General experience & gotchas (踩坑)
- **Cashback rate generally lower than Rakuten** (1-5% vs Rakuten's category-dependent 2-10%); use Rakuten when applicable (non-Amazon merchants) and Karma for Amazon + drop alerts.
- **Wishlist feature is the differentiator** — clean per-product price-drop alerts via email + push.
- **AI assistant feature** added 2025 — conversational "find me X" inside the extension; useful but treat its recommendations with hallucination skepticism (same as `domains/ai-shopping-assistants.md` guardrail).
- **Coverage**: Amazon, Best Buy, Walmart, Target, Macy's, Nike, Sephora, hundreds more.
- **Two-extension conflict** with Capital One Shopping / Honey / Rakuten — same affiliate-slot fight. If user already has Capital One Shopping, Karma adds value for wishlist; can co-exist if user accepts non-deterministic commission attribution.

## Failure signals & fallback
**Fallback:** Capital One Shopping (overlapping function) or Coupert (different coupon network).
